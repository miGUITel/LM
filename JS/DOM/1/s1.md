### Sesión 1 — JavaScript en la web y el DOM (texto didáctico)

Cuando hablamos de “programar en la web”, en realidad hablamos de repartir responsabilidades entre tres piezas que cooperan:

* **HTML**: define la **estructura**. Es el “esqueleto” del documento: qué hay (títulos, párrafos, listas, botones…).
* **CSS**: define la **apariencia**. Es la “piel”: colores, tamaños, márgenes, disposición.
* **JavaScript**: define el **comportamiento**. Es lo que hace que la página “reaccione”: cambia texto, añade o quita elementos, modifica estilos en función de lo que ocurre.

Un ejemplo de vida real: en una web de compras, el HTML contiene el carrito, el CSS lo hace bonito, y JavaScript actualiza el número de productos sin recargar la página.

---

## 1) El DOM: lo que realmente “toca” JavaScript

Una idea clave: **JavaScript no “edita el archivo HTML” como si fuera un Word**. Lo que hace es trabajar sobre una representación interna que el navegador construye al cargar la página: el **DOM**.

**DOM** significa *Document Object Model*. En la práctica:

* El navegador lee el HTML.
* Construye un **árbol de nodos** en memoria.
* Ese árbol representa el documento y sus partes.

### El documento como árbol de nodos

Imagina el HTML como una jerarquía:

* Hay un nodo raíz (el documento).
* De él cuelgan nodos hijos (html, head, body…).
* Dentro del body cuelgan más nodos (div, h1, p, ul, li…).

En el DOM encontramos principalmente dos tipos de nodos que nos interesan hoy:

* **Nodos de elemento**: etiquetas como `<p>`, `<div>`, `<button>`.
* **Nodos de texto**: el contenido textual dentro de una etiqueta.

### Relaciones padre–hijo

En un árbol:

* Un elemento puede tener **hijos** (por ejemplo, un `<ul>` tiene `<li>`).
* Un elemento puede tener **padre** (por ejemplo, un `<li>` tiene como padre a `<ul>`).
* También puede tener “hermanos” (otros `<li>` dentro del mismo `<ul>`).

Este modelo es muy importante porque explica por qué JavaScript puede:

* Localizar un elemento.
* Crear un elemento nuevo.
* Insertarlo “debajo” de otro.
* Eliminarlo del árbol.

---

## 2) El DOM es “en memoria”, no el HTML original

Esto evita una confusión típica del alumnado:

* El HTML es el **texto** del documento (lo que escribirías en un fichero `.html`).
* El DOM es la **estructura viva** que el navegador mantiene en memoria.

Cuando JavaScript cambia algo, lo que cambia es el DOM.
El resultado visual cambia “en pantalla”, pero el fichero original no se reescribe.

Por eso, cuando en una web aparece un nuevo mensaje o desaparece un bloque, no significa que “se haya reescrito el HTML en disco”; significa que JavaScript ha modificado el DOM.

---

## 3) Selección: si no puedo encontrarlo, no puedo tocarlo

La manipulación del DOM empieza casi siempre por la misma operación: **seleccionar** un elemento.

### `document.getElementById()`

* Se usa cuando el elemento tiene `id`.
* Devuelve **un único elemento** (o `null` si no existe).

```js
const titulo = document.getElementById("titulo");
```

**Idea importante:** el `id` debe ser único en la página. Si el alumnado repite el mismo `id` en dos elementos, el DOM deja de ser “fiable”.

### `document.querySelector()`

* Selecciona usando **selectores CSS**.
* Devuelve el **primer** elemento que coincide.

```js
const primerParrafo = document.querySelector("p");
const aviso = document.querySelector(".aviso");
const boton = document.querySelector("#btnCrear");
```

**Ventaja didáctica:** un único método permite seleccionar por etiqueta, clase o id, igual que en CSS.

### ¿Qué devuelve cada método?

Ambos devuelven un **elemento del DOM** (un “objeto elemento”), con propiedades y métodos:

* puedo cambiarle texto,
* puedo cambiarle atributos,
* puedo añadirle clases,
* puedo insertarlo en otro sitio,
* puedo eliminarlo.

Y aquí está el núcleo conceptual de la sesión:

> Si puedo seleccionar un elemento, puedo modificarlo.

---

## 4) Crear y eliminar: el DOM se construye y se “limpia”

Una página no tiene por qué ser estática. JavaScript puede crear elementos nuevos y colocarlos dentro del DOM.

### Crear un elemento: `document.createElement()`

Esto no lo pone aún en la página: solo lo crea “en la mano”.

```js
const item = document.createElement("li");
```

### Insertarlo: `appendChild()`

Para que aparezca, hay que insertarlo como hijo de otro elemento:

```js
lista.appendChild(item);
```

### Eliminar: `remove()`

Cuando ya no lo quiero, lo quito del árbol:

```js
item.remove();
```

Este ciclo (seleccionar → crear → insertar → eliminar) es exactamente lo que hacen muchas interfaces reales:

* chats (mensajes que se añaden),
* listas de tareas (items que se crean y se borran),
* avisos (aparecen y desaparecen).

# Demostración rápida: HTML + CSS + JS (mínimo y efectivo)

La siguiente mini-demo te sirve para enseñar en pocos minutos:

* Selección por `id` y por selector CSS.
* Modificación del DOM (texto).
* Creación de elementos (`createElement` + `appendChild`).
* Eliminación (`remove`).

## `index.html` [visitar](./index.html)

```html
<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>Demo DOM - Sesión 1</title>
  <link rel="stylesheet" href="style.css" />
</head>
<body>
  <main class="container">
    <h1 id="titulo">Demo: JavaScript modifica el DOM</h1>

    <p class="subtitulo">
      Esta página es estática… hasta que JavaScript empieza a cambiar el DOM.
    </p>

    <section class="panel">
      <h2>Zona de cambios</h2>
      <p id="estado" class="estado">Estado: listo</p>

      <div class="botonera">
        <button id="btnCambiarTexto">Cambiar texto</button>
        <button id="btnCrear">Crear elemento</button>
        <button id="btnEliminarUltimo">Eliminar último</button>
      </div>
    </section>

    <section class="panel">
      <h2>Lista dinámica</h2>
      <ul id="lista" class="lista">
        <li>Elemento inicial (HTML)</li>
      </ul>
    </section>
  </main>

  <script src="app.js"></script>
</body>
</html>
```

## `app.js`

```js
// Selección por ID
const titulo = document.getElementById("titulo");
const estado = document.getElementById("estado");
const lista = document.getElementById("lista");

// Selección por selector CSS
const subtitulo = document.querySelector(".subtitulo");

// Botones (selección por ID)
const btnCambiarTexto = document.getElementById("btnCambiarTexto");
const btnCrear = document.getElementById("btnCrear");
const btnEliminarUltimo = document.getElementById("btnEliminarUltimo");

let contador = 1;

btnCambiarTexto.addEventListener("click", () => {
  // Modificamos el DOM: texto de elementos existentes
  titulo.textContent = "El DOM ha sido modificado por JavaScript";
  subtitulo.textContent = "Fíjate: no hemos tocado el HTML en disco, solo el DOM en memoria.";
  estado.textContent = "Estado: texto actualizado";
});

btnCrear.addEventListener("click", () => {
  // Creamos un elemento nuevo
  const li = document.createElement("li");
  li.textContent = `Elemento creado con JS #${contador}`;
  contador++;

  // Lo insertamos en el DOM
  lista.appendChild(li);

  estado.textContent = "Estado: elemento creado y añadido a la lista";
});

btnEliminarUltimo.addEventListener("click", () => {
  // Eliminamos el último elemento de la lista (si existe)
  const ultimo = lista.querySelector("li:last-child");

  if (ultimo) {
    ultimo.remove();
    estado.textContent = "Estado: último elemento eliminado";
  } else {
    estado.textContent = "Estado: no hay elementos para eliminar";
  }
});
```

### Cómo usar la demo en clase (2–3 minutos)

1. Carga la página: “solo hay un elemento inicial”.
2. Pulsa **Cambiar texto**: “hemos seleccionado y modificado”.
3. Pulsa **Crear elemento** varias veces: “createElement + appendChild”.
4. Pulsa **Eliminar último**: “remove”.
5. Remata con la idea clave: **todo ha sido posible porque hemos sabido seleccionar**.
