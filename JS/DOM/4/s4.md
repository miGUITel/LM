### Sesión 4 — Modificación de estilos mediante JavaScript

**Objetivo conceptual:** entender que JavaScript puede **cambiar el aspecto visual** de la página en función del estado o de una acción del usuario, sin tocar directamente el CSS original.

---

## 1) Idea de partida: comportamiento frente a apariencia

Conviene comenzar la sesión recordando el reparto de responsabilidades:

* **HTML** define qué elementos existen.
* **CSS** define cómo se ven.
* **JavaScript** decide **cuándo** cambia ese aspecto.

La clave didáctica aquí es esta:

> JavaScript no sustituye a CSS: lo activa, lo desactiva o decide qué estilos aplicar en cada momento.

Esto es exactamente lo que ocurre en:

* menús que se despliegan,
* botones que cambian de color,
* avisos que se resaltan,
* mensajes de error o éxito.

---

## 2) Dos formas de modificar estilos con JavaScript

JavaScript puede modificar estilos de dos maneras. Es importante **compararlas**, pero dejar claro desde el principio cuál es la recomendable.

---

### 2.1. Modificar estilos en línea (`element.style`)

Cada elemento del DOM tiene una propiedad `style` que permite cambiar estilos directamente.

```js
aviso.style.backgroundColor = "lightyellow";
aviso.style.color = "black";
```

Aspectos importantes para el alumnado:

* Las propiedades CSS se escriben en **camelCase**:

  * `background-color` → `backgroundColor`
  * `font-size` → `fontSize`
* Estos estilos se aplican como **estilos en línea**.

Ventaja:

* Muy directo.
* Fácil de entender para empezar.

Limitación (conviene mencionarla):

* Mezcla comportamiento y apariencia.
* No es escalable cuando hay muchos estilos.

---

### 2.2. Modificar estilos mediante clases CSS (recomendado)

La forma más habitual y limpia de trabajar es **activar o desactivar clases CSS** desde JavaScript.

```js
aviso.classList.add("destacado");
aviso.classList.remove("destacado");
aviso.classList.toggle("destacado");
```

Aquí aparece un concepto muy importante: **`classList`**.

#### `classList`

Permite gestionar las clases de un elemento:

* `add()` → añade una clase
* `remove()` → elimina una clase
* `toggle()` → añade si no está / elimina si está

Ventajas claras:

* El CSS sigue controlando la apariencia.
* JavaScript solo decide el estado.
* El código es más claro y reutilizable.

> Idea clave: JavaScript cambia el estado, CSS se encarga de cómo se ve ese estado.

---

## 3) ¿Cuándo usar cada opción?

Para este nivel es suficiente con esta regla práctica:

* **Cambios puntuales y didácticos** → `style`
* **Cambios de estado** (activo, error, seleccionado, destacado…) → `classList`

No hace falta entrar en optimización ni rendimiento; lo importante es que el alumnado **entienda el criterio**.

---

## 4) Conexión con lo ya aprendido

Es muy importante que vean que **no aparece nada nuevo “aislado”**:

* Seguimos seleccionando elementos (`querySelector`).
* Seguimos reaccionando a eventos (`click`).
* Ahora, además de cambiar texto o crear elementos, **cambiamos su aspecto**.

Todo sigue el mismo patrón mental:

1. Seleccionar.
2. Decidir qué ocurre.
3. Modificar el DOM (contenido, estructura o estilos).

## `app.js`

```js
// Selección del aviso
const aviso = document.querySelector("#aviso");

// Botones
const btnNormal = document.querySelector("#btnNormal");
const btnDestacado = document.querySelector("#btnDestacado");
const btnError = document.querySelector("#btnError");

btnNormal.addEventListener("click", () => {
  // Quitamos todas las clases de estado
  aviso.classList.remove("destacado", "error");
});

btnDestacado.addEventListener("click", () => {
  aviso.classList.remove("error");
  aviso.classList.add("destacado");
});

btnError.addEventListener("click", () => {
  aviso.classList.remove("destacado");
  aviso.classList.add("error");
});
```

## Cómo cerrar la sesión en clase

Preguntas y reflexiones útiles:

* *¿Qué parte decide el color del aviso?* → CSS
* *¿Qué parte decide cuándo cambia?* → JavaScript
* *¿Qué pasa si quitamos el CSS pero dejamos el JS?*
* *¿Qué ocurre si dejamos el CSS pero no el JS?*

Cierre conceptual claro:

> JavaScript no “dibuja” la página: decide el estado.
> CSS interpreta ese estado y lo convierte en apariencia.

Con esto, el alumnado ya tiene todas las piezas necesarias para que en la **sesión 5** los eventos (click) actúen como disparadores completos de cambios en contenido, estructura y estilos del DOM.
